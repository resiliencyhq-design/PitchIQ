# Design System Catalogue v0.1

Status: Draft  
Sprint: PF-11  
Reference implementation: PitchIQ

## Purpose

This catalogue defines the first WellTrack/PitchIQ design-system baseline from the token, component and pattern audits.

## Design-system areas

| Area | Baseline | Maturity |
|---|---|---|
| Colour | Neon green/lime dark sports interface | Medium |
| Typography | Mobile-first expressive scale | Medium |
| Spacing | Layout-system seed exists | Medium |
| Radius | Four radius tokens exist | Medium |
| Elevation | Glass card shadow exists | Medium |
| Motion | Hard-coded but consistent enough to catalogue | Low-medium |
| Components | Button, card, marker, progress, nav candidates | Medium |
| Patterns | Landing, onboarding, position selection, home, training, results | Medium-high |
| Responsive | Mobile-first iPhone frame | Medium |

## Gaps

- Semantic token aliases needed.
- Motion tokens needed.
- Z-index tokens needed.
- Button and marker component specs needed.
- Accessibility rules need design-system inclusion.
- Design-system documentation should eventually move to the platform repo.
