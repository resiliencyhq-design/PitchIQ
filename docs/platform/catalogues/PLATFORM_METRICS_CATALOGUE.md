# Platform Metrics Catalogue v0.1

Status: Draft  
Sprint: PF-17  
Reference implementation: PitchIQ

## Purpose

This catalogue defines the first candidate metrics for platform governance and future HQ display.

## Metric groups

| Metric group | Candidate metrics |
|---|---|
| Product | MVP stage completion, onboarding completion, training flow readiness |
| Design system | token coverage, component reuse, pattern coverage |
| Engineering | module ownership, test coverage, build health, PR quality |
| Quality | accessibility, performance, release readiness, defects |
| Governance | ADR coverage, catalogue completeness, documentation freshness |
| Assets | canonical asset coverage, deprecated asset count, asset risk |
| Technical debt | open debt items, severity, remediation priority |
| HQ | audit status, platform health score, release status |

## Initial health indicators

1. Catalogue completeness percentage.
2. P1 ADR completion count.
3. Token coverage rating.
4. Component canonicalisation count.
5. Asset lifecycle coverage.
6. MVP stage completion.
7. Release checklist pass/fail.
8. iPhone smoke test pass/fail.
